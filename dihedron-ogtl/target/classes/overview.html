1. Abstract

The Strutlets project is an attempt to create a simple development framework for JSR-286 compliant portlets; it is heavily inspired by Struts2, from which it borrows part of its name (Struts2 + Portlet yields Strutlets).

2. Description

The strutlets library implements the MVC pattern the Struts2 way: it sports a single controller portlet that receives all incoming requests and translates them into action method invocations; then it retrieves the action result and based on an XML configuration file, on annotations or on educated guesses it decides which JSP will render the view part.

The framework emulates the interceptors logic of its senior counterpart, by providing an invocation process that flows through a stack of interceptors, each of which can work on the request and response objects, manipulate parameters, add value to the processing, perform additional tasks such as reporting or profiling.

3. How to use the framework

3.1 Setting up the portlet(s)

In order to set up the controller portlet, the following steps should be taken:

  1. add the strutlets-X.Y.Z.jar file and its dependencies to the project's WEB-INF/lib directory;
  2. set the action controller portlet as the portlet class in the portlet.xml, as follows: 
    		<portlet>
			<portlet-name>controller</portlet-name>
			<display-name>Demo Action Controller</display-name>
			<portlet-class>org.dihedron.strutlets.ActionController</portlet-class>
			[...]
		</portlet>
  3. specify the additional parameters, as detailed in section 3.2

The portlet is ready to be deployed even without specifying any parameter.
